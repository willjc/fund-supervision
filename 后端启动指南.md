# 后端启动指南

## 问题说明

如果在 `RuoYiApplication.java` 文件中看不到Run按钮,通常是以下原因:

1. **JDK版本不匹配**: 项目需要JDK 1.8,但你安装的是JDK 21
2. **IDE未识别为Maven项目**: 需要导入Maven项目
3. **Maven依赖未下载**: 需要下载项目依赖

---

## 环境要求

### 必需软件

| 软件 | 版本要求 | 当前项目 |
|------|----------|----------|
| **JDK** | 1.8 | ⚠️ 需要JDK 8 |
| **Maven** | 3.6+ | 推荐3.8+ |
| **MySQL** | 5.7+ / 8.0+ | 数据库 |
| **Redis** | 3.0+ | 缓存服务 |

### 当前问题

- ❌ 你安装的是 **JDK 21**
- ✅ 项目需要 **JDK 8** (Java 1.8)

---

## 解决方案

### 方案一: 安装JDK 8 (推荐)

#### 1. 下载JDK 8

**选项A: Oracle JDK 8**
- 下载地址: https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html
- 选择: `jdk-8u381-windows-x64.exe`

**选项B: OpenJDK 8 (推荐)**
- 下载地址: https://adoptium.net/temurin/releases/?version=8
- 选择: `OpenJDK 8 (LTS)` Windows x64 安装包

#### 2. 安装JDK 8

运行安装程序,建议安装到:
```
C:\Program Files\Java\jdk1.8.0_381
```

#### 3. 配置环境变量

**方法1: 系统环境变量**

1. 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
2. 新建系统变量:
   - 变量名: `JAVA_HOME`
   - 变量值: `C:\Program Files\Java\jdk1.8.0_381`
3. 编辑系统变量 `Path`,添加:
   ```
   %JAVA_HOME%\bin
   ```
4. 点击确定保存

**方法2: 仅为当前项目配置 (VSCode)**

在VSCode中,创建或修改 `.vscode/settings.json`:
```json
{
  "java.configuration.runtimes": [
    {
      "name": "JavaSE-1.8",
      "path": "C:\\Program Files\\Java\\jdk1.8.0_381",
      "default": true
    },
    {
      "name": "JavaSE-21",
      "path": "C:\\Program Files\\Java\\java-21"
    }
  ],
  "java.home": "C:\\Program Files\\Java\\jdk1.8.0_381"
}
```

#### 4. 验证安装

打开新的命令行窗口:
```bash
java -version
```

应该显示:
```
java version "1.8.0_381"
Java(TM) SE Runtime Environment (build 1.8.0_381-b09)
Java HotSpot(TM) 64-Bit Server VM (build 25.381-b09, mixed mode)
```

### 方案二: 升级项目到JDK 21 (不推荐)

如果坚持使用JDK 21,需要大量修改:

1. 修改 `pom.xml` 中的Java版本

4. 可能需要修改大量代码

**不推荐原因**:
- 工作量大,容易出错
- 可能导致不兼容问题
- 若依框架基于JDK 8开发

## 常见问题

### 问题1: 看不到Run按钮

**原因**:
- JDK版本不匹配
- IDE未识别项目
- Maven依赖未下载

**解决**:
1. 安装JDK 8
2. 重新导入Maven项目
3. 清理并重新加载Java语言服务器

### 问题2: 端口被占用

**错误**: `Port 8080 was already in use`

**解决**:
```bash
# 查找占用8080端口的进程
netstat -ano | findstr :8080

# 结束进程 (替换<PID>为实际进程号)
taskkill /F /PID <PID>
```

### 问题3: 数据库连接失败

**错误**: `Could not create connection to database server`

**检查**:
1. MySQL服务是否启动
2. 数据库名称是否正确
3. 用户名密码是否正确
4. 防火墙是否阻止连接

### 问题4: Redis连接失败

**错误**: `Unable to connect to Redis`

**解决**:
1. 启动Redis服务
2. 检查Redis配置
3. 测试连接: `redis-cli ping`

### 问题5: Maven依赖下载失败

**解决**:
```bash
# 清理Maven缓存
mvn clean

# 强制更新依赖
mvn clean install -U

# 配置国内镜像 (编辑 ~/.m2/settings.xml)
```

添加阿里云镜像:
```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

## 生产环境部署

### 打包部署

```bash
# 1. 清理并打包
mvn clean package

# 2. 上传jar到服务器
scp ruoyi-admin/target/ruoyi-admin.jar root@服务器IP:/opt/ruoyi/

# 3. 服务器上启动
nohup java -jar ruoyi-admin.jar > ruoyi.log 2>&1 &

# 4. 查看日志
tail -f ruoyi.log
```

### 配置为系统服务

创建 `/etc/systemd/system/ruoyi.service`:
```ini
[Unit]
Description=RuoYi Application
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/ruoyi
ExecStart=/usr/bin/java -jar /opt/ruoyi/ruoyi-admin.jar
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

启动服务:
```bash
systemctl daemon-reload
systemctl start ruoyi
systemctl enable ruoyi
systemctl status ruoyi
```
