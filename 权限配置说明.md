# 机构管理员权限配置说明

## 问题描述
机构管理员角色登录系统后,访问以下页面时提示"当前操作没有权限":
- 养老机构 → 机构管理 → 公示信息维护 (http://localhost/pension/institution/publicity)
- 养老机构 → 床位管理 → 床位列表 (http://localhost/pension/bed/list)

## 解决方案

### 方案一: 通过系统管理界面配置(推荐)

#### 第一步: 检查菜单是否存在
1. 使用管理员账号(admin)登录系统
2. 进入 **系统管理 → 菜单管理**
3. 搜索以下菜单,确认是否存在:
   - **公示信息维护** (路径: pension/institution/publicityManage)
   - **床位列表** (路径: pension/bed/list)

#### 第二步: 创建菜单(如果不存在)

**创建公示信息维护菜单:**
```
菜单名称: 公示信息维护
上级菜单: 机构管理
菜单类型: 菜单
显示排序: 4
路由地址: publicity
组件路径: pension/institution/publicityManage
权限字符: pension:publicity:list
菜单图标: guide
菜单状态: 正常
```

**为公示信息维护添加按钮权限:**
- 公示信息查询: `pension:publicity:query`
- 公示信息新增: `pension:publicity:add`
- 公示信息修改: `pension:publicity:edit`
- 公示信息删除: `pension:publicity:remove`
- 公示信息导出: `pension:publicity:export`
- 发布公示: `pension:publicity:publish`
- 取消公示: `pension:publicity:unpublish`

**创建床位列表菜单:**
```
菜单名称: 床位列表
上级菜单: 床位管理
菜单类型: 菜单
显示排序: 1
路由地址: list
组件路径: pension/bed/list
权限字符: elder:bed:list
菜单图标: list
菜单状态: 正常
```

**为床位列表添加按钮权限:**
- 床位查询: `elder:bed:query`
- 床位新增: `elder:bed:add`
- 床位修改: `elder:bed:edit`
- 床位删除: `elder:bed:remove`
- 床位导出: `elder:bed:export`
- 床位导入: `elder:bed:import`

#### 第三步: 为机构管理员角色分配权限
1. 进入 **系统管理 → 角色管理**
2. 找到 **机构管理员** 角色,点击"修改"
3. 在"菜单权限"树形结构中勾选:
   - ☑ 养老机构 (父菜单)
     - ☑ 机构管理 (父菜单)
       - ☑ 公示信息维护
         - ☑ 公示信息查询
         - ☑ 公示信息新增
         - ☑ 公示信息修改
         - ☑ 公示信息删除
         - ☑ 公示信息导出
         - ☑ 发布公示
         - ☑ 取消公示
     - ☑ 床位管理 (父菜单)
       - ☑ 床位列表
         - ☑ 床位查询
         - ☑ 床位新增
         - ☑ 床位修改
         - ☑ 床位删除
         - ☑ 床位导出
         - ☑ 床位导入
4. 点击"提交"保存

#### 第四步: 重新登录验证
1. 退出管理员账号
2. 使用机构管理员账号重新登录
3. 验证是否可以访问:
   - 养老机构 → 机构管理 → 公示信息维护
   - 养老机构 → 床位管理 → 床位列表

### 方案二: 通过SQL脚本配置(适合批量操作)

如果需要批量配置或在开发环境快速部署,可以执行SQL脚本:

1. 打开 `sql/add_institution_admin_permissions.sql` 文件
2. 按照文件中的说明,先查询机构管理员角色ID:
   ```sql
   SELECT role_id, role_name FROM sys_role WHERE role_name LIKE '%机构%';
   ```
3. 确认角色ID后,执行整个SQL脚本
4. 重新登录系统验证

## 注意事项

1. **数据权限**: 机构管理员通常只能查看和管理自己关联的机构数据,需要在角色管理中配置"数据范围"为"自定义数据权限"。

2. **菜单层级**: 确保所有父菜单都已勾选,否则子菜单不会显示。

3. **缓存刷新**: 修改权限后,建议:
   - 清除浏览器缓存
   - 重新登录系统
   - 如果使用Redis缓存,可能需要清除用户权限缓存

4. **权限字符规则**:
   - 模块:功能:操作
   - 例如: `pension:publicity:list` (养老:公示:列表)
   - 例如: `elder:bed:add` (养老:床位:新增)

## 常见问题

### Q1: 修改权限后仍然提示无权限?
**A**: 需要重新登录系统,让权限生效。若依框架在用户登录时会加载权限到缓存。

### Q2: 菜单显示了但点击后还是提示无权限?
**A**: 检查是否勾选了对应的按钮权限(如查询、新增、修改等)。

### Q3: 如何验证权限是否配置正确?
**A**: 可以在角色管理中查看该角色已分配的菜单和按钮权限列表。

### Q4: 机构管理员能看到其他机构的数据?
**A**: 需要在角色管理中配置"数据范围"为"自定义数据权限",并关联用户与机构的映射关系。
