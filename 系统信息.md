# 养老机构预收费资金监管平台 - 系统信息

## 一、系统概述

### 1.1 系统基本信息

- **系统名称**: 养老机构预收费资金监管平台
- **系统版本**: v1.0
- **开发框架**: 若依(RuoYi) v3.9.0
- **系统定位**: 养老机构预收费资金全流程监管,保障老年人权益
- **架构模式**: 四端协同(民政监管端 + 养老机构端 + H5小程序端 + 数据统计平台)
- **开发策略**: 单模块开发,功能导向,快速迭代

### 1.2 系统目标

本系统旨在通过信息化手段,实现对养老机构预收费资金的全流程监管,包括:
- 机构入驻审批和资质管理
- 老人入住和账户管理
- 订单支付和资金监管
- 押金使用和资金划拨
- 预警监控和风险防控
- 数据统计和决策支持

### 1.3 四端协同架构

**1. 民政监管端(PC)**:
- 机构审批、资金监管、预警核验
- 数据统计、决策支持

**2. 养老机构端(PC)**:
- 机构运营、入住管理、订单处理
- 押金申请、资金划拨

**3. H5小程序端(移动)**:
- 机构浏览、预约参观
- 订单支付、费用查询
- 押金申请、评价反馈

**4. 数据统计平台(PC)**:
- 数据大屏、预警监控
- 综合统计、可视化展示

## 四、数据库配置 ⭐重点

### 4.1 达梦数据库配置(DM8)

**数据库类型**: 达梦数据库 (DM8)

**驱动信息**:
- 驱动类: `dm.jdbc.driver.DmDriver`
- 连接地址格式: `jdbc:dm://[主机地址]:[端口]/[数据库名]`
- 连接示例: `jdbc:dm://localhost:5236/NEWZIJIN`

**数据库信息**:
- 数据库名: `NEWZIJIN`
- 字符集: UTF-8
- 默认端口: 5236

**Maven依赖配置**:
```xml
<dependency>
    <groupId>com.dameng</groupId>
    <artifactId>DmJdbcDriver18</artifactId>
    <version>8.1.2.192</version>
</dependency>
```

**Druid连接池配置**:
```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driverClassName: dm.jdbc.driver.DmDriver
    druid:
      master:
        url: jdbc:dm://localhost:5236/NEWZIJIN
        username: [用户名]
        password: [密码]
      # 连接池配置
      initialSize: 5
      minIdle: 10
      maxActive: 20
      maxWait: 60000
      timeBetweenEvictionRunsMillis: 60000
      minEvictableIdleTimeMillis: 300000
      maxEvictableIdleTimeMillis: 900000
      validationQuery: SELECT 1 FROM DUAL
      testWhileIdle: true
      testOnBorrow: false
      testOnReturn: false
      # 监控配置
      webStatFilter:
        enabled: true
      statViewServlet:
        enabled: true
        urlPattern: /druid/*
        loginUsername: yl
        loginPassword: 123456
      filter:
        stat:
          enabled: true
          logSlowSql: true
          slowSqlMillis: 1000
          mergeSql: true
        wall:
          config:
            multi-statement-allow: true
```

### 4.2 连接池配置说明

| 配置项 | 值 | 说明 |
|--------|-----|------|
| initialSize | 5 | 初始连接数 |
| minIdle | 10 | 最小空闲连接数 |
| maxActive | 20 | 最大活动连接数 |
| maxWait | 60000 | 获取连接最大等待时间(毫秒) |
| timeBetweenEvictionRunsMillis | 60000 | 空闲连接检测间隔(毫秒) |
| minEvictableIdleTimeMillis | 300000 | 最小空闲时间(毫秒) |
| maxEvictableIdleTimeMillis | 900000 | 最大空闲时间(毫秒) |
| validationQuery | SELECT 1 FROM DUAL | 连接验证SQL |
| testWhileIdle | true | 空闲时检测连接有效性 |
| testOnBorrow | false | 获取连接时不检测 |
| testOnReturn | false | 归还连接时不检测 |
| slowSqlMillis | 1000 | 慢SQL阈值(毫秒) |

## 八、安全配置

### 8.1 JWT Token配置

```yaml
token:
  header: Authorization
  secret: abcdefghijklmnopqrstuvwxyz
  expireTime: 999999  # 令牌有效期(分钟)
```

**配置说明**:
- **header**: Token在HTTP请求头中的字段名
- **secret**: Token加密密钥(生产环境建议使用更复杂的密钥)
- **expireTime**: Token有效期,单位分钟(约1.9年)

### 8.2 密码策略

```yaml
user:
  password:
    maxRetryCount: 99999  # 最大错误次数
    lockTime: 10  # 锁定时间(分钟)
```

**配置说明**:
- **maxRetryCount**: 密码输入错误最大次数,超过后锁定账户
- **lockTime**: 账户锁定时间,单位分钟

### 8.3 文件上传限制

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB  # 单个文件大小限制
      max-request-size: 20MB  # 总上传大小限制

file:
  path: /home/zijin/uploadPath  # 文件上传路径
```

**配置说明**:
- **max-file-size**: 单个文件最大大小
- **max-request-size**: 单次请求最大大小
- **path**: 文件上传保存路径

### 8.4 XSS防护

系统内置XSS过滤器,自动过滤以下内容:
- HTML标签
- JavaScript代码
- SQL注入关键字

### 8.5 CSRF防护

使用Spring Security的CSRF防护机制:
- 所有POST、PUT、DELETE请求需要CSRF Token
- Token存储在Session中
- 前端自动从Cookie中获取Token

```xml
<dependency>
```

### 10.2 SQL语法差异

| AUTO_INCREMENT | IDENTITY(1,1) | 自增主键 |
| IFNULL(expr1, expr2) | NVL(expr1, expr2) | 空值处理 |
| CONCAT(str1, str2) | str1 \|\| str2 | 字符串连接 |
| DATE_FORMAT(date, format) | TO_CHAR(date, format) | 日期格式化 |
| STR_TO_DATE(str, format) | TO_DATE(str, format) | 字符串转日期 |

- 押金超额预警
- 入驻超额预警
- 监管超额预警
- 风险保证金预警
- 大额支付预警

- 民政监管端: 机构审批、资金监管、预警核验

- 机构床位数不能超过备案床位数

- 首月服务费在入驻订单支付后自动扣除

## 十四、开发和构建

### 14.1 后端开发

```bash

cd ruoyi-ui

# 开发环境启动(端口: 80)
```

**H5端开发**:
```bash

# 开发环境启动(端口: 3000)
npm run dev

# 生产环境构建
npm run build

# 生成的静态文件位于
# ruoyi-h5/dist/
```

### 14.3 部署

**后端部署**:
```bash
# 1. 上传JAR文件到服务器
scp ruoyi-admin/target/ruoyi-admin.jar user@server:/opt/ruoyi/

# 2. 启动应用
java -jar /opt/ruoyi/ruoyi-admin.jar

# 3. 后台运行
nohup java -jar /opt/ruoyi/ruoyi-admin.jar > /opt/ruoyi/logs/app.log 2>&1 &

# 4. 使用systemd管理(推荐)
# 创建服务文件: /etc/systemd/system/ruoyi.service
# 启动服务: systemctl start ruoyi
# 开机自启: systemctl enable ruoyi
```

**前端部署**:
```bash
# 1. 构建静态文件
npm run build:prod

# 2. 上传到服务器
scp -r dist/* user@server:/usr/share/nginx/html/

# 3. 配置Nginx
# 编辑: /etc/nginx/conf.d/ruoyi.conf
# 重启Nginx: systemctl restart nginx
```

**Nginx配置示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 管理端
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # H5端
    location /h5 {
        alias /usr/share/nginx/html/h5;
        index index.html;
        try_files $uri $uri/ /h5/index.html;
    }

    # 后端API代理
    location /dev-api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 十五、常见问题

### 15.1 数据库连接失败

**问题**: 启动时提示数据库连接失败

**解决方案**:
1. 检查数据库是否启动: `systemctl status dm`
2. 检查数据库连接信息是否正确
3. 检查数据库用户权限
4. 检查防火墙是否开放5236端口

### 15.2 Redis连接失败

**问题**: 启动时提示Redis连接失败

**解决方案**:
1. 检查Redis是否启动: `systemctl status redis`
2. 检查Redis连接信息是否正确
3. 检查Redis密码配置
4. 检查防火墙是否开放6379端口

### 15.3 前端无法访问后端

**问题**: 前端调用后端接口失败

**解决方案**:
1. 检查后端是否启动
2. 检查前端API地址配置
3. 检查Nginx代理配置
4. 检查跨域配置
5. 检查防火墙规则

### 15.4 文件上传失败

**问题**: 上传文件时提示失败

**解决方案**:
1. 检查文件大小是否超过限制
2. 检查上传路径是否存在
3. 检查上传路径权限
4. 检查磁盘空间是否充足

### 15.5 登录后无权限

**问题**: 登录后看不到菜单或提示无权限

**解决方案**:
1. 检查用户角色是否分配
2. 检查角色菜单权限是否配置
3. 清除浏览器缓存
4. 清除Redis缓存
5. 重新登录
