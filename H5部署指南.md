# H5端部署指南

## 一、部署前准备

### 1.1 环境要求
- Node.js 14+
- npm 或 yarn
- 服务器环境: Nginx 或 Apache

### 1.2 已修改的配置文件

#### `.env.production` - 生产环境配置
```env
# 生产环境配置
NODE_ENV = 'production'

# H5接口地址
VUE_APP_BASE_API = 'http://jg.dayushaiwang.com'
```

**说明**:
- 后端API地址已配置为 `http://jg.dayushaiwang.com`
- 前端打包后会自动使用此地址

## 三、服务器部署

### 方式一: Nginx部署(推荐)

#### 3.1 上传文件到服务器
```bash
# 将 dist 目录上传到服务器
scp -r dist/* root@your-server:/var/www/h5/
```

#### 3.2 Nginx配置
在服务器上编辑Nginx配置:

```nginx
# /etc/nginx/conf.d/h5.conf
server {
    listen 80;
    server_name h5.dayushaiwang.com;  # 你的H5域名

    # 前端静态文件目录
    root /var/www/h5;
    index index.html;

    # 开启gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 反向代理API请求(可选,如果不想前端直接访问后端域名)
    # location /api {
    #     proxy_pass http://jg.dayushaiwang.com;
    #     proxy_set_header Host $host;
    #     proxy_set_header X-Real-IP $remote_addr;
    #     proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    #     proxy_set_header X-Forwarded-Proto $scheme;
    # }

    # 日志
    access_log /var/log/nginx/h5_access.log;
    error_log /var/log/nginx/h5_error.log;
}
```

#### 3.3 重启Nginx
```bash
# 测试配置
nginx -t

# 重启服务
systemctl restart nginx

# 或者重新加载配置
nginx -s reload
```

### 方式二: Apache部署

#### 3.1 上传文件
```bash
scp -r dist/* root@your-server:/var/www/html/h5/
```

#### 3.2 .htaccess配置
在 `/var/www/html/h5/` 目录创建 `.htaccess`:

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /h5/
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /h5/index.html [L]
</IfModule>

# 开启Gzip压缩
<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html text/plain text/xml text/css text/javascript application/javascript application/json
</IfModule>

# 静态资源缓存
<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/jpg "access plus 1 year"
  ExpiresByType image/jpeg "access plus 1 year"
  ExpiresByType image/gif "access plus 1 year"
  ExpiresByType image/png "access plus 1 year"
  ExpiresByType text/css "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
</IfModule>
```

## 四、跨域问题处理

### 4.1 后端配置CORS(推荐)
在后端 `RuoYiApplication.java` 或配置类中添加CORS支持:

```java
@Configuration
public class CorsConfig {
    @Bean
    public CorsFilter corsFilter() {
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowCredentials(true);

        // 允许的域名
        config.addAllowedOriginPattern("http://h5.dayushaiwang.com");
        config.addAllowedOriginPattern("http://localhost:8081");

        config.addAllowedHeader("*");
        config.addAllowedMethod("*");
        source.registerCorsConfiguration("/**", config);
        return new CorsFilter(source);
    }
}
```

### 4.2 Nginx反向代理(备选)
如果后端不支持CORS,可以用Nginx代理:

```nginx
location /api {
    proxy_pass http://jg.dayushaiwang.com;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;

    # 添加CORS头
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods 'GET, POST, PUT, DELETE, OPTIONS';
    add_header Access-Control-Allow-Headers 'DNT,X-Mx-ReqToken,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Authorization';

    if ($request_method = 'OPTIONS') {
        return 204;
    }
}
```

然后修改 `.env.production`:
```env
VUE_APP_BASE_API = '/api'
```

## 五、HTTPS配置(推荐)

### 5.1 申请SSL证书
- 使用Let's Encrypt免费证书
- 或者从阿里云/腾讯云购买证书

### 5.2 Nginx HTTPS配置
```nginx
server {
    listen 443 ssl http2;
    server_name h5.dayushaiwang.com;

    # SSL证书配置
    ssl_certificate /etc/nginx/ssl/h5.dayushaiwang.com.pem;
    ssl_certificate_key /etc/nginx/ssl/h5.dayushaiwang.com.key;
    ssl_session_timeout 5m;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers on;

    root /var/www/h5;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

# HTTP自动跳转HTTPS
server {
    listen 80;
    server_name h5.dayushaiwang.com;
    return 301 https://$server_name$request_uri;
}
```

### 5.3 更新API地址为HTTPS
如果后端也支持HTTPS,修改 `.env.production`:
```env
VUE_APP_BASE_API = 'https://jg.dayushaiwang.com'
```

## 六、部署检查清单

### 6.1 部署前检查
- [ ] 已修改 `.env.production` 中的API地址
- [ ] 已执行 `npm run build` 构建
- [ ] 检查 `dist/` 目录是否生成
- [ ] 检查后端API是否可访问: `http://jg.dayushaiwang.com`

### 6.2 部署后检查
- [ ] 访问H5页面是否正常加载
- [ ] 打开浏览器控制台,检查是否有报错
- [ ] 测试登录功能
- [ ] 测试API调用是否正常
- [ ] 检查静态资源是否加载成功
- [ ] 测试移动端适配效果

### 6.3 常见问题排查

#### 问题1: 页面404
**原因**: Nginx未配置SPA路由支持
**解决**: 添加 `try_files $uri $uri/ /index.html;`

#### 问题2: API请求失败
**原因**: 跨域或API地址配置错误
**解决**:
1. 检查 `.env.production` 配置
2. 检查后端CORS配置
3. 查看浏览器Network面板的请求详情

#### 问题3: 静态资源404
**原因**: publicPath配置问题
**解决**: 检查 `vue.config.js` 中 `publicPath` 配置

#### 问题4: 白屏
**原因**: JS加载失败或路由配置错误
**解决**:
1. 查看浏览器Console错误
2. 检查Nginx根目录配置
3. 检查文件权限

## 十、快速部署脚本

### 10.1 本地构建脚本
创建 `deploy.sh`:
```bash
#!/bin/bash
echo "开始构建H5项目..."

# 构建
npm run build

if [ $? -eq 0 ]; then
    echo "构建成功!"
    echo "输出目录: dist/"
    echo "请手动上传到服务器或运行上传脚本"
else
    echo "构建失败,请检查错误信息"
    exit 1
fi
```

### 10.2 自动上传脚本
创建 `upload.sh`:
```bash
#!/bin/bash
SERVER="root@your-server-ip"
REMOTE_PATH="/var/www/h5/"

echo "开始上传文件到服务器..."
scp -r dist/* $SERVER:$REMOTE_PATH

if [ $? -eq 0 ]; then
    echo "上传成功!"
    echo "请访问: http://h5.dayushaiwang.com"
else
    echo "上传失败"
    exit 1
fi
```

使用方式:
```bash
# 赋予执行权限
chmod +x deploy.sh upload.sh

# 执行部署
./deploy.sh && ./upload.sh
```
