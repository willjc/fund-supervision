# SQL脚本执行说明

## 文件: create_menus_and_permissions.sql

### 脚本功能
创建公示信息维护和床位列表的菜单及所有按钮权限,但不分配给任何角色。

### 执行前提条件
1. 确保数据库中已存在以下父菜单:
   - **机构管理** (menu_type = 'M')
   - **床位管理** (menu_type = 'M')

2. 如果父菜单不存在,需要先创建父菜单

### 执行步骤

#### 1. 连接数据库
```bash
mysql -u root -p
use newzijin;
```

#### 2. 执行SQL脚本
```bash
source d:/newhm/newzijin/sql/create_menus_and_permissions.sql
```

或者在MySQL客户端中直接复制粘贴SQL内容执行。

#### 3. 验证结果
执行完毕后,脚本会自动输出验证查询结果:
- 公示信息维护的所有权限列表
- 床位管理的所有权限列表
- 权限数量统计

预期结果:
- 公示信息维护: 8个权限 (1个菜单 + 7个按钮)
- 床位管理: 7个权限 (1个菜单 + 6个按钮)

#### 公示信息维护 (pension:publicity)

#### 床位列表 (elder:bed)

### 执行后操作

脚本执行完成后,需要在系统管理界面为角色分配权限:

1. 登录系统 (使用admin账号)
2. 进入 **系统管理 → 角色管理**
3. 找到 **机构管理员** 角色,点击"修改"
4. 在"菜单权限"中勾选刚创建的菜单和按钮
5. 点击"提交"保存
6. 让机构管理员重新登录系统

### 注意事项

1. **幂等性**: 脚本使用 `NOT EXISTS` 检查,可以重复执行而不会插入重复数据

2. **父菜单依赖**: 如果找不到父菜单,对应的子菜单将不会被创建,需要先创建父菜单

3. **权限标识唯一性**: 每个权限标识(perms)必须唯一,脚本已做去重处理

4. **排序**: 菜单按 order_num 排序,可以后续在系统管理中调整

5. **状态**: 所有菜单默认状态为"正常"(status='0'),可见性为"显示"(visible='0')

### 故障排查

#### 问题1: 父菜单不存在
**错误现象**: 变量为NULL,子菜单未创建

**解决方案**: 先创建父菜单
```sql
-- 查看现有菜单结构
SELECT menu_id, menu_name, parent_id, menu_type, perms
FROM sys_menu
WHERE menu_name IN ('养老机构', '机构管理', '床位管理')
ORDER BY menu_id;

-- 如果缺少,需要先创建父菜单
```

#### 问题2: 权限已存在
**错误现象**: 脚本跳过某些插入

**解决方案**: 正常现象,说明权限已存在,脚本避免重复插入

#### 问题3: 执行后在系统中看不到菜单
**可能原因**:
1. 父菜单不存在或未分配给当前角色
2. 浏览器缓存未刷新
3. 未重新登录系统

**解决方案**:
1. 检查父菜单是否存在并已分配
2. 清除浏览器缓存
3. 重新登录系统

### 回滚操作

如果需要删除创建的菜单和权限:

```sql
-- 删除公示信息维护的所有权限
DELETE FROM sys_menu WHERE perms LIKE 'pension:publicity:%';

-- 删除床位管理的所有权限
DELETE FROM sys_menu WHERE perms LIKE 'elder:bed:%';

-- 删除角色菜单关联
DELETE FROM sys_role_menu
WHERE menu_id NOT IN (SELECT menu_id FROM sys_menu);

COMMIT;
```
