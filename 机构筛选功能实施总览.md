# 养老机构筛选功能实施总览

## 一、功能需求回顾

根据用户需求截图,需要在H5端机构列表页实现以下筛选功能:

### H5端筛选维度
1. **床位情况**: 全部机构 / 有空位机构 / 免费试住机构
2. **区域筛选**: 郑州市12个区县选择
3. **机构性质**: 民办 / 公办 / 公建民营
4. **收住类型**: 自理 / 半护理 / 全护理 / 失能 / 失智 (支持多选)
5. **医疗条件**: 内设医疗机构 / 与医疗机构合作 / 自营医疗机构 / 无医养结合
6. **机构星级**: 1-5星
7. **总费用**: 价格区间筛选

### 后端管理需求
- 在机构入驻表单中添加对应字段的录入
- 区县选择使用标准化的郑州市区划数据

## 五、关键技术要点

### 1. 收住类型多选实现

**数据库存储**:
```sql
-- 存储为逗号分隔字符串
care_levels = "1,2,3"  -- 表示支持自理、半护理、全护理
```

**后端查询**:
```xml
<!-- MyBatis使用FIND_IN_SET进行多选匹配 -->
<if test="careLevels != null and careLevels != ''">
    and (
    <foreach collection="careLevels.split(',')" item="level" separator=" or ">
        FIND_IN_SET(#{level}, care_levels)
    </foreach>
    )
</if>
```

**前端处理**:
```javascript
// Vue中使用数组管理多选状态
careLevels: ['1', '2', '3']

// 提交时转为字符串
submitData.careLevels = form.careLevels.join(',')

// 回显时转为数组
form.careLevels = data.careLevels.split(',')
```

### 2. 价格区间筛选逻辑

**原理**: 判断两个区间是否有交集

```sql
-- 机构价格区间: [price_range_min, price_range_max]
-- 筛选价格区间: [priceRangeMin, priceRangeMax]
-- 有交集条件: 机构最高价 >= 筛选最低价 AND 机构最低价 <= 筛选最高价

WHERE price_range_max >= #{priceRangeMin}
  AND price_range_min <= #{priceRangeMax}
```

**示例**:
- 机构A: 1500-3000元
- 筛选: 2001-3000元
- 结果: 匹配(交集为2001-3000)
