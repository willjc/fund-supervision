## 第一章 项目基本情况

### 1.1 项目背景

随着我国人口老龄化进程加快，养老服务业快速发展。然而，部分养老机构存在预收费资金管理不规范、资金安全风险高等问题，老年人合法权益难以得到有效保障。为规范养老机构收费行为，防范资金风险，保障老年人合法权益，亟需建立一套完善的预收费资金监管平台。

### 1.2 建设目标

本项目旨在建设一个集"民政监管部门 + 养老机构 + 小程序端 + 数据统计平台"四端协同的养老机构预收费资金监管平台，实现：

1. **资金全流程监管**：对养老机构预收资金进行全程监控，确保资金安全
2. **风险预警机制**：建立多维度预警体系，及时发现和处置风险
3. **便民服务提升**：为老年人及家属提供便捷的缴费、查询、投诉等服务
4. **监管决策支持**：通过数据分析为政府部门提供决策依据

### 1.3 项目范围

本项目涵盖以下业务范围：

- 养老机构入驻审批与信息管理
- 老人入住登记与信息管理
- 床位资源管理
- 订单生成与支付管理
- 资金监管账户管理
- 押金使用申请与审批
- 资金划拨管理
- 预警规则配置与监控
- 数据统计与分析
- H5小程序便民服务

## 第三章 项目概述

### 3.1 系统架构

本系统采用前后端分离的微服务架构，四端协同设计：

```
┌─────────────────────────────────────────────────────────────────┐
│                        养老机构预收费资金监管平台                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ 民政监管端   │  │ 养老机构端   │  │ H5小程序端   │  │ 数据统计平台 │  │
│  │  (PC Web)   │  │  (PC Web)   │  │  (Mobile H5) │  │  (大屏展示)  │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  └──────────────┘  │
│           │                 │                 │                 │          │
│           └─────────────────┴─────────────────┴─────────────────┘          │
│                                  │                                     │
│                    ┌─────────────▼─────────────┐                          │
│                    │      后端API服务          │                          │
│                    │   (Spring Boot)          │                          │
│                    └─────────────┬─────────────┘                          │
│                                  │                                     │
│           ┌──────────────────────┼──────────────────────┐                │
│           ▼                      ▼                       ▼                │
│  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐          │
│  │   MySQL     │      │    Redis     │      │  银行接口    │          │
│  │  数据库      │      │   缓存        │      │  支付网关    │          │
│  └──────────────┘      └──────────────┘      └──────────────┘          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

系统当前使用**模拟支付**方式实现订单支付功能，用于演示和测试环境。

##### 模拟支付流程
1. 用户点击"立即支付"按钮
2. 系统显示支付确认对话框
3. 调用 `/h5/payment/process/{orderId}` 接口
4. 后端模拟1秒支付延迟
5. 自动返回支付成功
6. 更新订单状态为"已支付"
7. 自动创建老人账户（如不存在）
8. 根据订单金额分配账户余额
9. 扣除首月服务费（入驻订单）
10. 生成后续月份资金划拨单

##### 正式对接指南
如需对接真实支付网关，请参考以下步骤：

1. **准备支付渠道资源**
   - 申请支付渠道商户号（微信/支付宝/银联）
   - 获取API密钥和证书
   - 配置回调通知地址

2. **修改支付处理逻辑**
   - 文件位置: `ruoyi-admin/src/main/java/com/ruoyi/web/controller/h5/H5OrderController.java`
   - 方法: `processPayment()` (第875-1030行)

3. **集成支付SDK**
   - 添加支付渠道SDK依赖到pom.xml
   - 配置商户信息
   - 实现统一下单接口

4. **配置回调处理**
   - 实现支付结果异步通知处理
   - 验证回调签名
   - 更新订单和账户状态

5. **联调测试**
   - 沙箱环境测试
   - 生产环境小额测试
