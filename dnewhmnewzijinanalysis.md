# 两个老人管理页面的区别分析

## 1. 入住人列表 (http://localhost/pension/elder/list)
**文件**: ruoyi-ui/src/views/pension/elder/list.vue
**业务定位**: 养老机构管理自己的入住人
**权限范围**: 只能看到本机构的入住人
**主要功能**:
- 财务管理: 去支付、续费、退费、押金使用
- 信息管理: 详情、维护、家属管理
- 数据统计: 入住总人数、服务费余额、押金余额、会员卡余额
**操作按钮**: 去支付、详情、维护、续费、退费、押金使用、家属管理、删除
**特点**: 业务功能完整,包含财务和入住管理

## 2. 老人信息 (http://localhost/elder/info)
**文件**: ruoyi-ui/src/views/elder/info/index.vue
**业务定位**: 系统管理员管理所有老人基础信息
**权限范围**: 能看到所有机构的老人
**主要功能**:
- 基础信息管理: 新增、修改、删除、导出
- 关系管理: 家属管理、设置密码
- 入住申请: 为老人创建入住申请
**操作按钮**: 修改、删除、家属管理、设置密码、入住申请
**特点**: 侧重基础信息和关系管理,不涉及财务

## 3. 功能对比

| 功能 | 入住人列表 | 老人信息 |
|------|-----------|---------|
| 查看范围 | 本机构入住人 | 所有老人 |
| 财务管理 | ✓ (支付/续费/退费) | ✗ |
| 家属管理 | ✓ | ✓ |
| 设置密码 | ✗ (待添加) | ✓ |
| 入住申请 | ✗ | ✓ |
| 信息维护 | ✓ | ✓ |

## 4. 建议
入住人列表页面也应该添加"设置密码"按钮,因为:
1. 机构需要帮助入住老人设置H5登录密码
2. 家属需要通过H5端查看老人信息
3. 两个页面功能应该对等
