# H5端跨域配置说明

## 跨域问题说明

当H5前端部署在一个域名(如 http://h5.dayushaiwang.com),后端API部署在另一个域名(如 http://jg.dayushaiwang.com)时,浏览器的同源策略会阻止跨域请求,导致跨域错误。

**常见错误提示**:
```
Access to XMLHttpRequest at 'http://jg.dayushaiwang.com/h5/user/login'
from origin 'http://h5.dayushaiwang.com' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present on the requested resource.
```

---

## 解决方案一: 后端配置CORS (推荐) ✅

### 已完成的配置

**文件**: `ruoyi-framework/src/main/java/com/ruoyi/framework/config/ResourcesConfig.java`

**配置内容**:
```java
@Bean
public CorsFilter corsFilter()
{
    CorsConfiguration config = new CorsConfiguration();
    // 设置允许跨域的域名(允许所有域名)
    config.addAllowedOriginPattern("*");
    // 设置访问源请求头
    config.addAllowedHeader("*");
    // 设置访问源请求方法
    config.addAllowedMethod("*");
    // 是否允许发送Cookie
    config.setAllowCredentials(true);
    // 有效期 1800秒
    config.setMaxAge(1800L);
    // 添加映射路径，拦截一切请求
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return new CorsFilter(source);
}
```

### 配置说明

- `addAllowedOriginPattern("*")`: 允许所有域名访问
- `addAllowedHeader("*")`: 允许所有请求头
- `addAllowedMethod("*")`: 允许所有HTTP方法(GET、POST、PUT、DELETE等)
- `setAllowCredentials(true)`: **重要!** 允许携带Cookie和Authorization头
- `setMaxAge(1800L)`: 预检请求缓存时间30分钟

### 如何生效

1. **重新编译项目**:
```bash
cd ruoyi-admin
mvn clean package
```

2. **重启后端服务**:
```bash
# 如果是直接运行jar
java -jar ruoyi-admin.jar

# 如果是使用systemd等服务管理
systemctl restart ruoyi
```

3. **测试跨域**:
   - 访问H5页面
   - 打开浏览器开发者工具(F12)
   - 查看Network面板,检查API请求是否成功
   - 查看Response Headers中是否有CORS相关头

### 预期的Response Headers

请求成功后,后端响应头应该包含:
```
Access-Control-Allow-Origin: http://h5.dayushaiwang.com
Access-Control-Allow-Credentials: true
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers: *
```

## 解决方案二: 限制特定域名 (更安全)

如果你不想允许所有域名访问,可以修改为只允许特定域名:

```java
@Bean
public CorsFilter corsFilter()
{
    CorsConfiguration config = new CorsConfiguration();

    // 只允许特定域名访问
    config.addAllowedOrigin("http://h5.dayushaiwang.com");
    config.addAllowedOrigin("http://localhost:8081"); // 开发环境
    config.addAllowedOrigin("https://h5.dayushaiwang.com"); // HTTPS

    config.addAllowedHeader("*");
    config.addAllowedMethod("*");
    config.setAllowCredentials(true);
    config.setMaxAge(1800L);

    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return new CorsFilter(source);
}
```

**注意**: 使用 `addAllowedOrigin()` 时,不能使用通配符 `*`,必须指定具体域名。

## 解决方案三: Nginx反向代理 (备选)

如果不方便修改后端代码,可以使用Nginx反向代理。

### 3.1 Nginx配置

编辑H5的Nginx配置文件:

```nginx
server {
    listen 80;
    server_name h5.dayushaiwang.com;
    root /var/www/h5;
    index index.html;

    # 前端页面
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 反向代理API请求
    location /api/ {
        # 代理到后端
        proxy_pass http://jg.dayushaiwang.com/;

        # 传递真实IP
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 添加CORS头
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, PUT, DELETE, OPTIONS';
        add_header Access-Control-Allow-Headers 'DNT,X-Mx-ReqToken,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Authorization';
        add_header Access-Control-Allow-Credentials 'true';

        # 处理OPTIONS预检请求
        if ($request_method = 'OPTIONS') {
            return 204;
        }
    }
}
```

### 3.2 修改前端API地址

修改 `ruoyi-h5/.env.production`:

```env
# 原配置
VUE_APP_BASE_API = 'http://jg.dayushaiwang.com'

# 改为
VUE_APP_BASE_API = '/api'
```

### 3.3 重新构建和部署

```bash
cd ruoyi-h5
npm run build
scp -r dist/* root@服务器IP:/var/www/h5/
```

### 3.4 重启Nginx

```bash
nginx -t
systemctl restart nginx
```

## 常见问题排查

### 问题1: 仍然报CORS错误

**可能原因**:
1. 后端代码未重新编译部署
2. 后端服务未重启
3. 浏览器缓存

**解决方法**:
```bash
# 1. 重新编译后端
cd ruoyi-admin
mvn clean package

# 2. 重启后端服务
systemctl restart ruoyi

# 3. 清除浏览器缓存
Ctrl + Shift + Delete
```

### 问题2: OPTIONS预检请求失败

**现象**: 看到两个请求,第一个OPTIONS请求失败

**原因**: 浏览器在发送跨域POST/PUT/DELETE请求前,会先发送OPTIONS预检请求

**解决**: 确保后端CORS配置已生效,允许OPTIONS方法

### 问题3: 携带Cookie失败

**现象**: 请求发送成功,但Cookie或Authorization头未携带

**原因**: `setAllowCredentials(true)` 未配置

**解决**: 确保后端配置了 `config.setAllowCredentials(true);`

### 问题4: 只有部分接口跨域

**原因**: 可能某些接口使用了自定义的响应处理

**解决**: 检查是否有拦截器或过滤器覆盖了CORS设置

## 生产环境建议

### 1. 限制允许的域名

生产环境建议不使用 `*` 允许所有域名,而是明确指定:

```java
config.addAllowedOrigin("https://h5.dayushaiwang.com");
```

### 2. 使用HTTPS

配置HTTPS证书,提升安全性:
- 前端: https://h5.dayushaiwang.com
- 后端: https://jg.dayushaiwang.com

### 3. 配置CDN

如果使用CDN,确保CDN也配置了正确的CORS头。

### 4. 监控CORS错误

在前端添加错误监控,及时发现CORS问题:

```javascript
// 在 request.js 中
service.interceptors.response.use(
  response => {
    // ...
  },
  error => {
    if (error.message.includes('CORS')) {
      console.error('CORS错误:', error)
      // 上报到监控系统
    }
    return Promise.reject(error)
  }
)
```
