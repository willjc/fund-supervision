## 1. 系统概述

### 1.1 项目简介

养老机构预收费资金监管平台是一套基于若依（RuoYi）框架开��的全流程资金监管系统，旨在保障老年人预收费资金安全，规范养老机构经营行为。

**系统特点**：
- 四端协同：民政监管端 + 养老机构端 + 小程序端（H5） + 数据统计平台
- 资金托管：银行监管账户模式，资金按规则划拨
- 实时监控：多维度预警规则，风险及时发现
- 数据公开：小程序端可查询费用余额和资金流水

## 2. 环境准备

### 2.1 开发环境配置

#### 2.1.1 JDK 1.8 安装配置

**Windows 环境：**

1. 下载 JDK 1.8（推荐 Oracle JDK 或 OpenJDK 8）
2. 安装 JDK 到指定目录，如 `C:\Program Files\Java\jdk1.8.0_xxx`
3. 配置环境变量：

```batch
# 新建系统变量
JAVA_HOME = C:\Program Files\Java\jdk1.8.0_xxx

# 编辑 Path 变量，添加
%JAVA_HOME%\bin
```

4. 验证安装：

```batch
java -version
# 输出示例：
# java version "1.8.0_xxx"
# Java(TM) SE Runtime Environment (build 1.8.0_xxx-xxx)
```

**macOS 环境：**

```bash
# 使用 Homebrew 安装
brew install openjdk@8

# 配置环境变量（添加到 ~/.zshrc 或 ~/.bash_profile）
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 1.8)' >> ~/.zshrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 验证安装
java -version
```

**Linux 环境：**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-8-jdk

# CentOS/RHEL
sudo yum install java-1.8.0-openjdk-devel

# 验证安装
java -version
```

#### 2.1.2 Node.js 16.x/18.x 安装配置

**Windows 环境：**

1. 访问 [Node.js官网](https://nodejs.org/) 下载 LTS 版本安装包

#### 2.1.3 Maven 3.6+ 安装配置

**Windows 环境：**

1. 下载 [Maven](https://maven.apache.org/download.cgi)
2. 解压到目录，如 `C:\apache-maven-3.9.x`
3. 配置环境变量：

```batch
# 新建系统变量
MAVEN_HOME = C:\apache-maven-3.9.x

# 编辑 Path 变量，添加
%MAVEN_HOME%\bin
```

4. 配置镜像（修改 `conf/settings.xml`）：

```xml
<mirrors>
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>*</mirrorOf>
    <name>阿里云公共仓库</name>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

5. 验证安装：

```batch
mvn -v
# 输出示例：
# Apache Maven 3.9.x
# Java version: 1.8.0_xxx
```

**macOS/Linux 环境：**

```bash
# 使用 Homebrew 安装
brew install maven

# 或手动安装
wget https://dlcdn.apache.org/maven/maven-3/3.9.5/binaries/apache-maven-3.9.5-bin.tar.gz
tar -xzf apache-maven-3.9.5-bin.tar.gz
sudo mv apache-maven-3.9.5 /opt/maven

# 配置环境变量
echo 'export MAVEN_HOME=/opt/maven' >> ~/.zshrc
echo 'export PATH="$MAVEN_HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 验证安装
mvn -v
```

#### 2.1.4 MySQL 5.7+/8.0 安装配置

**Windows 环境：**

1. 下载 [MySQL Installer](https://dev.mysql.com/downloads/installer/)
2. 运行安装程序，选择安装 MySQL Server
3. 配置 root 密码，选择默认字符集为 `utf8mb4`
4. 验证安装：

```batch
mysql --version
# 输出示例：mysql  Ver 8.0.xx for Win64 on x86_64
```

**macOS 环境：**

```bash
# 使用 Homebrew 安装
brew install mysql@8.0

# 启动 MySQL 服务
brew services start mysql@8.0

# 设置 root 密码
mysql_secure_installation

# 验证安装
mysql --version
```

**Linux 环境：**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# CentOS/RHEL
sudo yum install mysql-server

# 启动服务
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码（MySQL 8.0）
sudo grep 'temporary password' /var/log/mysqld.log

# 登录并修改密码
mysql -u root -p
```

**配置 MySQL（所有平台）：**

编辑 MySQL 配置文件 `my.cnf`（Windows 为 `my.ini`）：

```ini
[mysqld]
# 设置默认字符集
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

# 设置时区
default-time-zone='+8:00'

# SQL 模式
sql_mode='STRICT_TRANS_TABLES,NO_ZERO_DATE,NO_ZERO_IN_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_ENGINE_SUBSTITUTION'

# 最大连接数
max_connections=500
```

## 3. 数据库部署

### 3.1 创建数据库和用户

**登录 MySQL：**

```bash
mysql -u root -p
```

**创建数据库：**

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS newzijin DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用数据库用户（生产环境推荐）
CREATE USER 'zijinuser'@'localhost' IDENTIFIED BY '你的密码';
CREATE USER 'zijinuser'@'%' IDENTIFIED BY '你的密码';

-- 授权
GRANT ALL PRIVILEGES ON newzijin.* TO 'zijinuser'@'localhost';
GRANT ALL PRIVILEGES ON newzijin.* TO 'zijinuser'@'%';

-- 刷新权限
FLUSH PRIVILEGES;

-- 查看数据库
SHOW DATABASES;

-- 退出
EXIT;
```

### 4.3 服务启动脚本

**创建启动脚本 `start.sh`：**

```bash
#!/bin/bash

# 设置Java环境变量
export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH

# 设置应用目录
APP_HOME=/home/zijin/fund-supervision
APP_NAME=ruoyi-admin
JAR_NAME=$APP_HOME/ruoyi-admin/target/$APP_NAME.jar

# 设置JVM参数
JAVA_OPTS="-Xms512m -Xmx1024m -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=256m"

# 设置配置文件路径
CONFIG_OPTS="--spring.config.location=$APP_HOME/ruoyi-admin/src/main/resources/"

# 设置日志输出
LOG_DIR=$APP_HOME/logs

# 创建日志目录
mkdir -p $LOG_DIR

echo "================================"
echo "  启动养老资金监管平台后端服务"
echo "================================"
echo ""

# 检查JAR文件是否存在
if [ ! -f "$JAR_NAME" ]; then
    echo "错误: JAR文件不存在: $JAR_NAME"
    exit 1
fi

# 检查是否已运行
PID=$(ps -ef | grep $JAR_NAME | grep -v grep | awk '{print $2}')
if [ -n "$PID" ]; then
    echo "警告: 服务已在运行，PID: $PID"
    echo "如需重启，请先执行 stop.sh"
    exit 1
fi

# 启动服务
echo "正在启动服务..."
nohup java $JAVA_OPTS -jar $JAR_NAME $CONFIG_OPTS > $LOG_DIR/application.log 2>&1 &

# 等待服务启动
sleep 5

# 检查服务状态
PID=$(ps -ef | grep $JAR_NAME | grep -v grep | awk '{print $2}')
if [ -n "$PID" ]; then
    echo "服务启动成功！"
    echo "PID: $PID"
    echo "日志文件: $LOG_DIR/application.log"
    echo ""
    echo "访问地址:"
    echo "  - 后端API: http://localhost:8080"
    echo "  - Swagger文档: http://localhost:8080/swagger-ui/"
    echo "  - Druid监控: http://localhost:8080/druid/"
else
    echo "服务启动失败，请查看日志"
    exit 1
fi

echo "================================"
```

**创建停止脚本 `stop.sh`：**

```bash
#!/bin/bash

APP_NAME=ruoyi-admin
JAR_NAME=/home/zijin/fund-supervision/ruoyi-admin/target/$APP_NAME.jar

echo "================================"
echo "  停止养老资金监管平台后端服务"
echo "================================"
echo ""

# 查找进程
PID=$(ps -ef | grep $JAR_NAME | grep -v grep | awk '{print $2}')

if [ -z "$PID" ]; then
    echo "服务未运行"
    exit 0
fi

echo "正在停止服务 (PID: $PID)..."
kill -15 $PID

# 等待进程结束
for i in {1..30}; do
    PID=$(ps -ef | grep $JAR_NAME | grep -v grep | awk '{print $2}')
    if [ -z "$PID" ]; then
        echo "服务已停止"
        exit 0
    fi
    sleep 1
done

# 强制结束
echo "强制停止服务..."
kill -9 $PID
echo "服务已停止"
```

```bash
#!/bin/bash

# 备份目录
BACKUP_DIR="/backup/zijin"

echo "[2/3] 备份上传文件..."

echo "上传文件备份完成: upload_$DATE.tar.gz"

# 备份配置文件
echo "[3/3] 备份配置文件..."
tar -czf $BACKUP_DIR/config_$DATE.tar.gz /home/zijin/fund-supervision/ruoyi-admin/src/main/resources/*.yml
echo "配置文件备份完成: config_$DATE.tar.gz"

echo ""
echo "================================"
echo "  备份完成！"
echo "================================"
echo ""

# 清理30天前的备份
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete
echo "已清理30天前的旧备份"
```

### 8.4 常见问题排查

#### 问题1：后端服务无法启动

**排查步骤：**

```bash
# 1. 检查Java环境
java -version

# 2. 检查端口是否被占用
netstat -tlnp | grep 8080

# 3. 检查配置文件
cat /home/zijin/fund-supervision/ruoyi-admin/src/main/resources/application-druid.yml

# 4. 查看日志
tail -f /home/zijin/logs/application.log

# 5. 测试数据库连接
mysql -u zijinuser -p -h localhost newzijin

# 6. 测试Redis连接
redis-cli ping
```

#### 问题2：前端页面404

**排查步骤：**

```bash
# 1. 检查Nginx配置
sudo nginx -t

# 2. 检查文件是否存在
ls -la /var/www/html/admin/

# 3. 检查Nginx日志
sudo tail -f /var/log/nginx/error.log

# 4. 检查文件权限
sudo chown -R www-data:www-data /var/www/html/
sudo chmod -R 755 /var/www/html/
```

#### 问题3：API接口跨域错误

**解决方案：**

1. 检查后端跨域配置是否启用
2. 检查Nginx代理配置
3. 确认前端API地址配置正确

#### 问题4：数据库连接失败

**排查步骤：**

```bash
# 1. 检查MySQL服务状态
sudo systemctl status mysqld

# 2. 检查数据库用户权限
mysql -u root -p
mysql> SELECT user, host FROM mysql.user WHERE user = 'zijinuser';

# 3. 检查防火墙
sudo firewall-cmd --list-all

# 4. 测试连接
mysql -u zijinuser -p -h localhost newzijin
```

#### 问题5：Redis连接失败

**排查步骤：**

```bash
# 1. 检查Redis服务状态
sudo systemctl status redis

# 2. 检查Redis配置
sudo cat /etc/redis/redis.conf | grep bind

# 3. 测试连接
redis-cli -h localhost -p 6379 ping

# 4. 检查防火墙
sudo firewall-cmd --list-ports
```
