## 12. 注意事项

### 12.1 认证方式
所有H5接口(除登录接口)都需要在请求头中携带Token:
```
Authorization: Bearer {token}
```

### 12.2 分页格式
统一使用若依框架的分页格式:
```json
{
  "code": 200,
  "msg": "操作成功",
  "rows": [],
  "total": 100
}
```

### 12.3 响应格式
统一使用若依框架的响应格式:
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

### 12.4 数据权限
- H5端用户只能查看自己关联的老人数据
- 通过 `elder_family` 表关联用户与老人
- 所有老人相关接口需要过滤数据权限

### 12.5 身份证脱敏
前端需要展示时,身份证号中间位需要脱敏显示:
- 原始: 410123199001011234
- 脱敏: 410***********1234

### 12.6 金额格式
- 数据库存储: Decimal(10,2)
- 接口传输: 数字类型,保留2位小数
- 前端显示: 格式化为货币格式 (¥1,234.56)
