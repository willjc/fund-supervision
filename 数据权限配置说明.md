# 养老机构系统 - 数据权限配置说明

## 概述

本系统通过`sys_user_institution`表实现机构用户的数据权限隔离,确保每个机构用户只能看到自己机构的数据。

## 数据权限表结构

### sys_user_institution 表

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | bigint(20) | 用户ID(主键) |
| institution_id | bigint(20) | 机构ID(主键) |
| create_time | datetime | 创建时间 |

**说明**:
- 一个用户可以关联多个机构(复合主键允许)
- 未在此表中配置关联的用户将看不到任何机构数据

## 配置步骤

### 1. 创建机构用户

在后台管理系统中:
1. 进入: 系统管理 -> 用户管理
2. 点击"新增"
3. 填写用户信息,分配机构角色
4. 保存用户

### 2. 配置机构关联

#### 方法一: 通过SQL直接配置(推荐)

```sql
-- 查看所有机构列表
SELECT institution_id, institution_name FROM pension_institution;

-- 查看需要配置的用户
SELECT user_id, user_name, nick_name FROM sys_user WHERE user_name = '用户名';

-- 为用户添加机构关联
INSERT INTO sys_user_institution (user_id, institution_id)
VALUES (用户ID, 机构ID);

-- 示例: 为用户ID=101的用户关联机构ID=1
INSERT INTO sys_user_institution (user_id, institution_id)
VALUES (101, 1);
```

#### 方法二: 批量配置

```sql
-- 为所有未配置的机构角色用户自动关联机构
-- (假设用户名包含机构信息)
INSERT INTO sys_user_institution (user_id, institution_id)
SELECT
    u.user_id,
    pi.institution_id
FROM sys_user u
INNER JOIN sys_user_role ur ON u.user_id = ur.user_id
INNER JOIN sys_role r ON ur.role_id = r.role_id
INNER JOIN pension_institution pi ON /* 根据实际情况关联 */
WHERE r.role_name LIKE '%机构%'
AND NOT EXISTS (
    SELECT 1 FROM sys_user_institution ui
    WHERE ui.user_id = u.user_id
);
```

### 3. 验证配置

```sql
-- 查看用户的机构关联
SELECT
    u.user_id,
    u.user_name,
    u.nick_name,
    ui.institution_id,
    pi.institution_name
FROM sys_user u
INNER JOIN sys_user_institution ui ON u.user_id = ui.user_id
INNER JOIN pension_institution pi ON ui.institution_id = pi.institution_id
WHERE u.user_name = '用户名';
```

## 常见问题

### Q1: 机构用户登录后看不到任何数据?

**原因**: 用户未配置机构关联

**解决**:
```sql
-- 检查用户是否有机构关联
SELECT * FROM sys_user_institution WHERE user_id = 用户ID;

-- 如果没有记录,添加关联
INSERT INTO sys_user_institution (user_id, institution_id)
VALUES (用户ID, 机构ID);
```

### Q2: 机构用户能看到所有机构的数据?

**原因**:
1. Controller未设置`currentUserId`
2. Mapper XML未添加数据权限过滤条件

**解决**: 参考`BedInfoController`和`PensionResidentController`的实现

### Q3: 如何让一个用户管理多个机构?

**解决**:
```sql
-- 为同一用户添加多个机构关联
INSERT INTO sys_user_institution (user_id, institution_id) VALUES
(用户ID, 机构ID1),
(用户ID, 机构ID2),
(用户ID, 机构ID3);
```

### Q4: 如何删除错误的机构关联?

**解决**:
```sql
DELETE FROM sys_user_institution
WHERE user_id = 用户ID AND institution_id = 机构ID;
```

### Q5: 管理员能看到所有机构的数据吗?

**解决**:
- 管理员用户(如admin)不设置机构关联
- 在查询时不传入`currentUserId`,或者在代码中判断是否为管理员
- 管理员角色可以看到所有数据

## 实现模板

### Controller层
```java
@GetMapping("/list")
public TableDataInfo list(YourVO queryVO)
{
    startPage();
    // 数据权限过滤: 只查询当前用户关联的机构的数据
    queryVO.setCurrentUserId(getUserId());
    List<YourVO> list = yourService.selectList(queryVO);
    return getDataTable(list);
}
```

### VO层
```java
/** 当前用户ID(用于数据权限过滤) */
private Long currentUserId;

public Long getCurrentUserId() {
    return currentUserId;
}

public void setCurrentUserId(Long currentUserId) {
    this.currentUserId = currentUserId;
}
```

### Mapper XML层
```xml
<select id="selectYourList" parameterType="YourVO" resultMap="YourResult">
    SELECT * FROM your_table t
    LEFT JOIN bed_info bi ON t.bed_id = bi.bed_id
    WHERE 1=1
    <!-- 数据权限过滤 -->
    <if test="currentUserId != null">
        AND bi.institution_id IN (
            SELECT institution_id FROM sys_user_institution WHERE user_id = #{currentUserId}
        )
    </if>
    <!-- 其他查询条件 -->
</select>
```
